Solución · Gobierno del Dato

4 Pilares del Gobierno del Dato Maestro

01
Estándares GS1
Normas universales de codificación GTIN, SSCC y GLN. Relación técnica con entes reguladores.
02
Flujo de Aprobación
Ningún SKU se activa sin aprobación de I&D. Criterio: viabilidad logística y técnica.
03
Score de Integridad
Cada SKU recibe un puntaje de completitud. SKUs incompletos no replican a sistemas downstream.
04
Mejora Continua
Dashboards de calidad del dato, alertas automáticas y revisión periódica de políticas.

Regla de oro: El Score Mínimo de Integridad determina si un SKU puede ser publicado en E-Commerce, procesado por Leafio, y operado en WMS/TMS. Un SKU sin dimensiones logísticas = bloqueado para distribución.

Propuesta Técnica · Implementación

¿Dónde crear el MDM Hub?

Vive en la misma infraestructura SQL Server de SAP B1 — como una base de datos de gobierno independiente, sin romper ni tocar las BDs fuente.

🗄️

SBD_MDM_MASTER

Nueva BD en el mismo SQL Server. Contiene el Golden Record de cada SKU con todos los campos de las 3 BDs unificados y validados.

📋 Tabla: MDM_SKU_MASTER (Golden Record)
📋 Tabla: MDM_SKU_SCORE (Score Integridad)
📋 Tabla: MDM_AUDIT_LOG (Trazabilidad)
📋 Tabla: MDM_APPROVAL_FLOW (Aprobaciones)
⚙️

Capa ETL / Sync

Proceso de integración que extrae, transforma y carga los campos de cada BD hacia el Hub y redistribuye el Golden Record enriquecido.

🔄 SQL Server Integration Services (SSIS)
🔄 SQL Agent Jobs (sincronización programada)
🔄 Stored Procedures + Triggers de validación
🔄 Python / API REST para distribución externa
👥

¿Quién trabaja dónde?

Cada área mantiene su acceso actual en SAP B1. Solo I&D actúa como custodio del MDM Hub y aprueba cambios al Golden Record.

🏢 Compras / Holding → SAP B1 HOLDING (sin cambios)
📦 Logística / CDD → SAP B1 CDD (sin cambios)
🛒 E-Commerce → consume Golden Record vía API
🔬 I&D (tú) → administra y gobierna SBD_MDM_MASTER

🔑 Principio clave: Las BDs de SAP B1 existentes no se modifican ni se tocan. El MDM Hub lee de ellas via linked servers o vistas, consolida el Golden Record y lo publica. Cero disrupciones operacionales.

E-Commerce · Desafíos de Data

E-Commerce: Jurisprudencia y Enriquecimiento

Hoy el e-commerce consume datos directos de la base de datos, pero carece de la capa deriquecimiento necesaria para una experiencia de usuario premium.

⚠️

El Vacío del "Empaque"

Faltan descripciones técnicas y comerciales diferenciadas por nivel de empaque:

📦 Empaque 1 (Unidad): Descripciones comerciales, fotos de detalle, beneficios de uso.
📦 Empaque 2 (Caja/Inner): Dimensiones logísticas, peso bruto, unidades por caja, EAN-14.
🔍

Riesgo de Jurisprudencia Directa

Conectar el e-commerce directamente a la DB transaccional sin un MDM expone:

  • Inconsistencias si se mofica un campo en el ERP.
  • Nombres técnicos de SAP que no son aptos para el cliente final.
  • Falta de validación previa (Score de Integridad).
  • Dificultad para enriquecer campos SEO/Marketing que no existen en SAP.

Solución: El MDM Hub permitirá que el área comercial enriquezca estas descripciones de empaque una sola vez y se distribuyan validadas a la web.
